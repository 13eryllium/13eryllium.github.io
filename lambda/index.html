Scripting for GameMaker Studio 2

A lightweight, embeddable scripting language for GMS2.

GMS2 v1.2.0 .lam

Lambda is a dynamically-typed scripting language designed to run inside GameMaker Studio 2 games, the same way Lua powers modding in other engines.

What Lambda is not: Lambda does not replace GML. It runs alongside it. Lambda scripts are sandboxed guests that interact only through the functions and constants you register.
Getting Started

Quick Start#

Set up Lambda running in your project in minutes.

1. Create an environment

var env = lambda_env();
lambda_register_stdlib(env);

2. Expose your game to mod scripts

// Expose a function
lambda_register_function(env, "spawn", function(layer, instance) {
    instance_create_layer(layer, instance, "Instances", obj_enemy);
});

// Expose a static constant
lambda_register_constant(env, "TILE_SIZE", 32);

// Expose a live game value (re-read on every access)
lambda_register_dynamic_constant(env, "player_x", function() {
    return obj_player.x;
});

3. Run a script

// From a .lam file bundled in Included Files
lambda_run_file(env, "mod.lam");

// Or from a string
lambda_run(env, "print($\"player is at {player_x}\");");

4. Call Lambda functions from GML

// Grab as a reusable GML callable
var on_update = lambda_grab_func(env, "on_update");
on_update(delta_time);

// Or one-shot call by name
lambda_call(env, "on_hit", other.id, damage);

5. Clean up

lambda_env_destroy(env);
Getting Started

Installation#

Lambda ships as a GMS2 extension. Add the .yymps to your project, then place .lam files in Included Files so they're bundled with your game build.

VS Code Extension: Full editor support is available - syntax highlighting, real-time linting for undeclared/unused variables, auto-formatting, and minification.
Language

Variables#

All variables are declared with let. Lambda is dynamically typed - a variable can hold any value type.

let x = 10;
let name = "hello";
let flag = true;
let nothing = null;

Variables are scoped to the block they're declared in. There is no hoisting.

Language

Functions & Closures#

Functions are created with the func keyword and assigned to variables.

let add = func(a, b) {
    return a + b;
};

let result = add(3, 4); // 7

Higher-order functions

let apply = func(f, x) { return f(x); };

let double = func(n) { return n * 2; };
apply(double, 5); // 10

Closures

Functions capture their enclosing scope. This enables patterns like counters and factories.

let make_counter = func() {
    let n = 0;
    return func() {
        n += 1;
        return n;
    };
};

let counter = make_counter();
counter(); // 1
counter(); // 2
counter(); // 3
Language

Control Flow#

If / else

if (x > 5) {
    print("big");
} else if (x == 3) {
    print("three");
} else {
    print("small");
}

While loop

let x = 5;
while (x > 0) {
    print(x);
    x -= 1;
}

C-style for loop

for (let i = 0; i < 10; i += 1) {
    print(i);
}

For-in loop

Iterate over every element of an array.

let items = ["sword", "shield", "potion"];
for (let item in items) {
    print(item);
}

Break & Continue

for (let i = 0; i < 10; i += 1) {
    if (i == 3) { continue; }
    if (i == 7) { break; }
    print(i);
}
Language

Arrays#

Arrays are zero-indexed and dynamically sized.

let arr = [1, 2, 3];

arr[0] = 99;
let len = array_length(arr); // 3

array_push(arr, 4);
array_pop(arr); // removes and returns 4

// Functional operations
let doubled = array_map(arr, func(v) { return v * 2; });
let evens   = array_filter(arr, func(v) { return v % 2 == 0; });
let sum     = array_reduce(arr, func(acc, v) { return acc + v; }, 0);
Language

Structs#

Structs are key-value containers. Access fields with dot notation or bracket notation.

let pt = { x: 10, y: 20 };

pt.x = 50;
pt["y"] = 80;

struct_has(pt, "x");  // true
struct_keys(pt);      // ["x", "y"]
struct_delete(pt, "y");

// Merging (override wins on conflict)
let base     = { hp: 100, speed: 5 };
let override = { speed: 8, name: "knight" };
let unit = struct_merge(base, override);
// { hp: 100, speed: 8, name: "knight" }
Internal note: Struct keys are prefixed with _ in GML to avoid reserved-word collisions. This is fully transparent inside Lambda - you never see the prefix.

Chained access

let val = game.entities[0].name;
Language

Pointers#

The >=> operator creates a live reference to a struct field or array index. Reading the pointer variable automatically dereferences it - you always get the current value, not a snapshot.

let pos = { x: 0, y: 0 };
let px >=> pos.x;

pos.x = 5;
print(px);  // 5  - reflects live state

pos.x = 99;
print(px);  // 99

Array pointer

let arr = [10, 20, 30];
let p1 >=> arr[1];

arr[1] = 99;
print(p1);  // 99
Language

String Interpolation#

Prefix a string literal with $ to embed expressions inside {}.

let name = "World";
print($"Hello, {name}!");        // Hello, World!

let a = 3;
let b = 4;
print($"{a} + {b} = {a + b}");  // 3 + 4 = 7

// Any expression works inside {}
print($"Type: {type_of(name)}");
Language

Wait Statement#

The wait statement defers a block of code to run after a number of seconds.

wait 3 once {
    run();
}
// run() is called approximately 3 seconds later.

Syntax

wait expr once {
    // body
}

expr is evaluated immediately and may be any number (decimals allowed). The body is compiled into an anonymous chunk at compile time and executes in the scope that was live when the wait ran.

Host integration

The host must call lambda_env_tick every Step event for waits to fire:

// Step event or controller object
lambda_env_tick(env);

Without this call, pending waits will never execute.

LAMBDA_DEBUG#

The LAMBDA_DEBUG macro in lambda.gml controls the verbosity of runtime and compile-time error messages. It defaults to false.

#macro LAMBDA_DEBUG false
  • false: Short, production-friendly error messages.
  • true: Extended context added at every pipeline stage.

When enabled, each stage enriches its errors as follows:

  • Lexer — character ordinal value
  • Parser — token index; got-value in all expect/unexpected errors
  • Compiler — chunk name in every thrown message
  • VM — type and value of offending operands; ip and chunk name for unknown opcodes

Set to true during development and false before shipping.

Error Handling#

try {
    throw "something went wrong";
} catch (err) {
    print($"Caught: {err}");
}

// Throw any value
try {
    throw { code: 404, msg: "not found" };
} catch (err) {
    print(err.msg);
}

// Assert (throws on false)
assert(x > 0, "x must be positive");

Host API Overview#

The Host API is the GML-side interface - the set of functions you call from your game to create environments, register names, and execute Lambda scripts.

Environment
lambda_env() lambda_env_destroy(env)
Registration
lambda_register_function(env, name, fn) lambda_register_constant(env, name, val) lambda_register_dynamic_constant(env, name, getter) lambda_register_stdlib(env)
Execution
lambda_run(env, source) lambda_run_file(env, path) lambda_precompile(env, source) lambda_chunk_run(env, chunk)
Calling Lambda
lambda_grab_func(env, name) lambda_call(env, name, arg0, ...)
Host API

Environment#

An environment is an isolated execution context. Scripts running in separate environments cannot see each other's variables or registered names.

lambda_env() → env struct

Creates and returns a new Lambda environment. Always call lambda_register_stdlib after creation unless you want a fully bare sandbox.

lambda_env_destroy(env)

Destroys the environment and frees all associated memory. Call this when the environment is no longer needed - e.g., when a level ends or a mod is unloaded.

var env = lambda_env();
lambda_register_stdlib(env);
// ... use env ...
lambda_env_destroy(env);
Host API

Execution#

lambda_run(env, source_string) → value

Compiles and runs a Lambda source string. Returns the last evaluated value.

lambda_run_file(env, path) → value

Loads and runs a .lam file.

lambda_precompile(env, source_string) → chunk

Compiles a source string into a reusable chunk. Faster than re-parsing the same script repeatedly.

lambda_chunk_run(env, chunk) → value

Executes a pre-compiled chunk.

var chunk = lambda_precompile(env, "player_x + 10");
// Call every frame cheaply:
var result = lambda_chunk_run(env, chunk);
lambda_grab_func(env, name) → GML callable

Returns a Lambda-defined function as a native GML callable. Use this to call into Lambda from your game loop without string lookups each time.

var on_update = lambda_grab_func(env, "on_update");
// In your Step event:
on_update(delta_time);
lambda_call(env, name, arg0, arg1, ...) → value

Calls a Lambda function by name with arguments. Convenient for one-off calls.

lambda_call(env, "on_hit", other.id, damage);
